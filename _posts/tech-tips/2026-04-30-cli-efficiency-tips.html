上周同事看我操作终端，问我：你怎么敲得这么快？

我说：不是手速快，是技巧省时间。

这篇文章分享7个我每天都在用的终端技巧。每个都很简单，但叠加起来，效率提升肉眼可见。

技巧一：Ctrl+R 搜索历史命令

这是我用得最多的技巧。以前敲过一条命令，下次要再用，不用重新输入，按Ctrl+R，输入关键词，自动匹配历史命令。

比如昨天我运行过：

docker exec -it mycontainer bash

今天再要进入容器，不需要重新敲。按Ctrl+R，输入"dock exec"，自动补全出来。回车，完事。

更进一步，按Ctrl+R可以连续匹配多个结果，按上下键在匹配结果中切换。

这个技巧让我省了大量重复敲命令的时间。

技巧二：!! 和 !$ 的妙用

!! 表示上一条命令。常用场景：

# 忘了加sudo
$ apt update
E: Could not open lock file...

# 不用重新敲，直接
$ sudo !!
sudo apt update

!$ 表示上一条命令的最后一个参数：

$ mkdir -p /very/long/path/to/project

$ cd !$
cd /very/long/path/to/project

这两个技巧，让我少敲了很多重复的字符。

技巧三：Ctrl+A/E 快速跳转行首行尾

命令敲到一半发现开头写错了，怎么改？

以前我的做法是按左箭头，一下一下往回跳。命令长了要按十几下。

现在：

  • Ctrl+A：跳到行首
  • Ctrl+E：跳到行尾

一行命令，瞬间跳到开头或结尾，不用一下一下按了。

技巧四：Ctrl+U/K 快速删除

命令敲错了想重来，以前一直按退格键。现在：

  • Ctrl+U：删除光标到行首的内容
  • Ctrl+K：删除光标到行尾的内容

想清空整行，先跳到行首（Ctrl+A），再删除到行尾（Ctrl+K）。

这三个快捷键（A/E/U/K）的组合，让我处理长命令时快了很多。

技巧五：Tab补全增强

Tab补全大家都知道，但很多人不知道还可以增强。

在.bashrc或.zshrc中加入：

# bash
bind 'set show-all-if-ambiguous on'

# zsh
autoload -Uz compinit && compinit

这样按Tab会显示所有匹配项，而不是只补全第一个。

更进阶的是zsh的自动补全插件，输入命令时实时提示可能的补全选项，不用等按Tab。很多人用Oh My Zsh就是为了这个。

技巧六：alias 把长命令变短

有些命令天天敲，每次都一长串，烦不烦？

我的做法是给常用命令设置别名：

# 常用alias
alias ll='ls -lah'
alias dc='docker-compose'
alias dps='docker ps --format "table {{.Names}}\t{{.Status}}\t{{.Ports}}"'
alias gst='git status'
alias glog='git log --oneline --graph --decorate -n 20'

这些别名写在.bashrc或.zshrc里，每次打开终端自动加载。

关键是：不要设置太多别名，只给真正常用的命令设置。设置太多反而记不住。

技巧七：命令组合的 ; && ||

以前我经常这样操作：

$ make build
# 等待执行完成
$ make test
# 等待执行完成
$ make deploy

现在一行搞定：

$ make build && make test && make deploy

三个连接符的区别：

  • ;：顺序执行，不管前一条是否成功
  • &&：前一条成功才执行下一条
  • ||：前一条失败才执行下一条

常用场景：

# 构建成功才部署
$ npm run build && npm run deploy

# 构建失败就清理临时文件
$ npm run build || rm -rf dist/

补充：这些技巧的前提

以上技巧主要适用于Bash和Zsh。如果你用的是Fish或其他Shell，快捷键可能不同。

我的建议：先熟练掌握一个Shell，把效率提升上来。换Shell的成本比学几个技巧大得多。

最后说一句

这7个技巧，每个单独看都很简单。但组合起来，每天能省下大量时间。

更重要的是，这些技巧让你专注于想做的事，而不是被命令行本身分散注意力。

如果你还没有用这些技巧，建议从Ctrl+R和Ctrl+A/E开始，这两个用得最多，效果也最明显。
