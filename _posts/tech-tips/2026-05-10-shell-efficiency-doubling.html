你有没有注意过，为什么有些人的终端五颜六色、信息一目了然，而你的还是黑白文本、挤成一团？

不是因为人家装了什么复杂配置，而是他们换掉了三个最基础的命令。

这篇文章不教你配置什么炫酷主题，就说三件事：把bash换成zsh，把cat换成bat，把ls换成eza。三个替换，终端体验直接跨代。

第一刀：zsh 换掉 bash

说实话，bash 够用了。但够用不等于好用。

zsh 多了什么？自动补全、历史搜索、路径展开、插件生态。这些东西看起来是小功能，但用熟之后，你会发现自己少敲了很多重复的东西。

最关键的是 Oh My Zsh。这个框架把 zsh 的配置难度从"要花一上午研究"降到了"一条命令装完就能用"。

# macOS
brew install zsh
sh -c "$(curl -fsSL https://raw.github.com/ohmyzsh/ohmyzsh/master/tools/install.sh)"

# Ubuntu/Debian
sudo apt install zsh
sh -c "$(curl -fsSL https://raw.github.com/ohmyzsh/ohmyzsh/master/tools/install.sh)"

装完重启终端，你会看到命令提示符变了。这就是 Oh My Zsh 默认主题 robbyrussell。已经有 git 分支显示、当前路径、状态标识了。

插件怎么开？编辑 ~/.zshrc，找到 plugins= 这一行：

plugins=(git z zsh-autosuggestions zsh-syntax-highlighting)

几个我常用的插件：

  • git：一堆 git 快捷别名，gst、gco、gp 这些
  • z：目录跳转神器，敲过一次的路径，下次用 z 关键词直接跳
  • zsh-autosuggestions：根据历史自动补全建议，按右箭头接受
  • zsh-syntax-highlighting：命令输错变红，输对变绿，不用回车就知道对不对

后面两个插件要单独装，但装完就忘不掉命令了。

第二刀：bat 换掉 cat

cat 这个命令的名字来自 concatenate（拼接），本来是用来拼接文件的。但我们大多数时候用它干什么？看文件内容。

问题是 cat 输出就是纯文本。代码没高亮，日志没颜色，行号也没有。一百行的配置文件，想定位第50行，还得自己数。

bat 做了什么？语法高亮 + 行号 + Git 变更标记 + 自动分页。

# macOS
brew install bat

# Ubuntu/Debian
sudo apt install bat

装完直接用：

bat config.yaml

你会看到：

  • 左侧是行号
  • 不同语法高亮显示（yaml、json、代码都支持）
  • 如果文件在 git 仓库里，被修改的行会有标记
  • 文件超过一屏会自动进 pager，空格翻页，q 退出

查看日志的时候尤其好用。以前看日志要 tail -f | grep --color，现在直接 tail -f /var/log/app.log | bat -l log --paging=never，日志自带颜色分列。

我常用的几个参数：

# 只看行号和内容，不要其他装饰
bat -p file.txt

# 显示所有字符，包括非打印字符
bat -A /etc/hosts

# 作为 man 的 pager，man 手册也带颜色
export MANPAGER="bat -plman"

说实话，换成 bat 之后，我几乎再也没用过 cat。除非是脚本里要拼接文件，那时候 bat 会自动检测非交互环境，降级成纯文本输出。

第三刀：eza 换掉 ls

ls 用了几十年，没什么问题。但你有想过文件列表可以更好看、更有信息量吗？

eza（读作"艾扎"）是 ls 的现代替代，用 Rust 写的。前身是 exa，后来 exa 停止维护了，eza 继承并继续开发。

# macOS
brew install eza

# Ubuntu/Debian
sudo apt install eza

直接 eza 看一眼：

  • 文件类型用颜色区分（目录、可执行、符号链接...）
  • 加 --icons 显示文件图标，一眼认出文件类型
  • 加 --git 显示每个文件的 git 状态
  • 加 -l 显示详细信息，权限、大小、修改时间

最实用的组合：

# 长列表 + git 状态 + 图标
eza -l --git --icons

# 树状显示，一目了然项目结构
eza --tree --level=2

# 只看目录
eza -d */

建议在配置文件里加 alias：

# ~/.zshrc 或 ~/.bashrc
alias ls='eza'
alias ll='eza -l --git --icons'
alias lt='eza --tree --level=2 --icons'
alias la='eza -la --git --icons'

这样 ls 自动用 eza，习惯不用改，体验直接升级。

注意：--icons 需要 Nerd Font 字体支持，否则图标会显示成方块。macOS 的 iTerm2 或 Terminal.app 装 Nerd Font 后要在配置里选一下。JetBrains Mono Nerd Font 是个不错的选择。

三刀砍完，还剩什么？

有人会说：装这么多东西，值得吗？

我的看法是：工具升级的成本是一次性的，效率提升的收益是每一天的。

装这三个东西，加起来不超过10分钟。但之后每一天，你都在用更好的版本。看文件有高亮，列目录有图标，敲命令有补全——这些细节叠加起来，一年省下来的时间和少犯的错误，远超那10分钟。

更重要的是，这些替换是无痛的：

  • zsh 完全兼容 bash 语法
  • bat 在非交互环境自动降级成纯文本
  • eza 的参数和 ls 几乎一致

没有学习成本，只有收益。

还有什么值得换的？

我试过一些其他"现代替代品"：

  • fd 换 find —— 好用，但 find 太万能了，有些复杂场景还是得用 find
  • ripgrep (rg) 换 grep —— 确实快，中文搜索也更友好，推荐
  • fzf —— 模糊搜索神器，和 bat、zsh 配合起来威力巨大

但我不推荐一上来就全换。先把 zsh、bat、eza 用顺，再根据需要扩展。

工具不是越多越好，是越顺手越好。

最后一句

换终端工具不像换 IDE、换框架那样是个大工程。就是装三个命令，改两行配置。

但这一小步，是从"能凑合用"到"每天顺手"的跨越。

如果你的终端还是十几年前的样子，不妨花10分钟换一把。敲命令的时间占工程师工作时间的大头，这一刀，砍得值。
