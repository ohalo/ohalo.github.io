想在飞牛NAS上搭建自己的磁力搜索引擎？这篇教程手把手教你，从零到完成不超过15分钟。

一、Bitmagnet是什么？

简单说：一个能自动爬取全球种子资源的搜索引擎。

它通过DHT网络24/7不间断爬取，你不需要去各种种子网站找资源——资源会自动"送上门"。

核心功能：

  • ✅ DHT爬虫：主动爬取全网种子信息
  • ✅ 智能分类：自动识别电影、电视剧、音乐、软件
  • ✅ Web界面：浏览器直接搜索
  • ✅ 套件集成：支持Sonarr、Radarr等媒体库

二、前置条件

项目要求
飞牛NAS系统fnOS 1.0+
Docker已启用
存储空间至少50GB（数据会持续增长）
网络稳定连接（DHT爬虫需要）

三、部署步骤

Step 1：创建文件夹

在飞牛NAS的文件管理中，进入 /docker 目录，创建 bitmagnet 文件夹。

路径：/vol1/docker/bitmagnet（根据你的实际卷号调整）

Step 2：打开Docker Compose

飞牛NAS已内置Docker Compose功能：

  1. 打开应用中心 → Docker
  2. 点击Compose → 新建项目
  3. 项目名称填写：bitmagnet

Step 3：粘贴配置文件

将以下配置粘贴到编辑框中：

version: '3.8'

services:
  bitmagnet:
    image: ghcr.io/bitmagnet-io/bitmagnet:latest
    container_name: bitmagnet
    ports:
      - "3333:3333"
      - "3334:3334/tcp"
      - "3334:3334/udp"
    restart: unless-stopped
    environment:
      POSTGRES_HOST: postgres
      POSTGRES_USER: postgres
      POSTGRES_PASSWORD: postgres
    command:
      - worker
      - run
      - --keys=http_server
      - --keys=queue_server
      - --keys=dht_crawler
    depends_on:
      postgres:
        condition: service_healthy

  postgres:
    image: postgres:16-alpine
    container_name: bitmagnet-postgres
    volumes:
      - ./data:/var/lib/postgresql/data
    restart: unless-stopped
    environment:
      POSTGRES_PASSWORD: postgres
      POSTGRES_DB: bitmagnet
      POSTGRES_USER: postgres
    shm_size: 1g
    healthcheck:
      test: ["CMD-SHELL", "pg_isready -U postgres"]
      interval: 10s
      timeout: 5s
      retries: 5
      start_period: 20s

配置说明：

  • 3333端口：Web界面和API
  • 3334端口：BitTorrent协议通信
  • dht_crawler：DHT爬虫（核心功能，不要删除）

Step 4：启动容器

点击创建，等待镜像下载和容器启动。

首次启动需要拉取两个镜像：

  • bitmagnet：主程序
  • postgres:16-alpine：数据库

下载时间取决于网络速度，通常3-5分钟。

Step 5：验证运行状态

容器启动后，在Docker容器列表中检查：

  • bitmagnet 状态应为 运行中
  • bitmagnet-postgres 状态应为 运行中

打开浏览器访问：http://你的NAS_IP:3333

如果看到Bitmagnet的搜索界面，说明部署成功！

四、首次运行注意事项

1. 数据库初始化

首次启动时，PostgreSQL需要初始化数据库，大约需要20-30秒。

如果bitmagnet容器反复重启，检查postgres容器是否正常运行。

2. DHT爬虫启动

进入Web界面后，点击右上角的三角形图标查看服务状态：

  • ✅ http_server：Web服务
  • ✅ queue_server：任务队列
  • ✅ dht_crawler：DHT爬虫

如果dht_crawler显示关闭状态，重启一下容器即可。

3. 数据开始爬取

DHT爬虫启动后，大约1-2分钟就能看到第一批种子出现在数据库中。

建议保持容器长期运行，运行时间越长，积累的种子越多。

五、常见问题

Q1：容器启动后Web界面打不开？

可能原因：

  • 端口被占用：检查3333端口是否被其他服务使用
  • 防火墙：确保飞牛NAS防火墙放行了3333端口

Q2：PostgreSQL健康检查失败？

解决方案：

  • 检查./data目录是否有写入权限
  • 尝试删除./data目录后重新创建容器

Q3：搜索结果为空？

原因：DHT爬虫刚启动，需要时间积累数据。

解决：耐心等待，通常运行24小时后就能搜到大量资源。

Q4：如何备份数据？

只需备份./data目录即可，这是PostgreSQL的数据目录。

六、与媒体库集成

Bitmagnet支持Torznab协议，可以与Prowlarr、Sonarr、Radarr等套件集成。

添加到Prowlarr

  1. 打开Prowlarr → Indexers → Add Indexer
  2. 搜索 generic torznab
  3. URL填写：http://NAS_IP:3333/torznab
  4. 点击Test测试连接
  5. 显示绿色✓后，点击Save

完成后，Sonarr/Radarr就能自动从Bitmagnet搜索资源了。

七、进阶配置（可选）

1. TMDB刮削

如果想让Bitmagnet自动识别电影/电视剧信息，可以配置TMDB API Key：

  1. 注册 The Movie Database 账号
  2. 在设置中获取API Key
  3. 在docker-compose中添加：TMDB_API_KEY: 你的key

2. 性能优化

  • 使用SSD存储数据库，查询更快
  • 调整shm_size值（默认1g，可增加到2g）
  • 定期清理无用的种子记录

八、安全提醒

  • ⚠️ Bitmagnet会主动参与DHT网络，建议配置防火墙规则
  • ⚠️ 考虑使用VPN增加匿名性
  • ⚠️ 请遵守版权法规，合理使用
