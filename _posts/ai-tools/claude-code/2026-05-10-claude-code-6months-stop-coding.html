去年底 Claude Code 发布的时候，我大概是第一波冲上去的人。终端里敲一行指令，AI 直接帮你改文件、跑测试、提交代码——这画面太诱人了。我当时心想：终于不用再跟 Copilot 那种一个补全一个补全地喂了，这玩意儿是真能自己干活。

半年过去了，我现在几乎不用它写代码了。

不是因为它烂。恰恰是因为它太好用了，好用到让我产生了一种错觉——觉得可以完全依赖它。直到被现实反复打脸，我才慢慢想明白它到底该放在什么位置。

先说它真正厉害的地方

我到现在都觉得，Claude Code 做项目初始化这件事，几乎没有对手。

你给它一段需求，它能在两三分钟内给你搭出一个完整的项目骨架：目录结构、配置文件、入口代码、甚至 Dockerfile 都给你写好。我自己从零搭一个 Next.js 项目，至少得半小时——还得翻文档确认各种配置项。Claude Code 三分钟搞定，而且基本上能跑。

还有一件事它特别擅长：写那种你自己懒得写但又不复杂的代码。比如给一堆数据写个转换脚本，给 API 加个新的 endpoint，给组件加几个 props——这种活它干得又快又准。你甚至不需要多想，直接说"帮我加个 XX 功能"，它就哐哐哐给你改完了。

写测试也是。单元测试、集成测试这种事，人类写着无聊，AI 写着不累。你让它给一个模块补测试，它能把边界情况都考虑到，比我自己写还全。

说白了，Claude Code 最强的场景是「明确需求 + 有限范围」。你告诉它要什么，边界在哪，它就能给你一个不错的结果。

然后说说让我放弃的部分

上下文丢得让我崩溃

这是我放弃的最大原因。

Claude Code 的上下文窗口再大，在真实项目里也不够用。我的项目几千个文件，它读着读着就把之前理解的东西忘了。最典型的场景：它刚改完 A 文件，过五分钟改 B 文件的时候，就把 A 文件里刚做的约定忘干净了。然后 A 和 B 之间的接口就对不上了。

你跟它说"记住我们刚才约定用 camelCase"，它能记住三分钟。三分钟后它又开始写 snake_case。这不是夸张，是真的反复发生。

更气人的是，它丢失上下文的方式是无声的。它不会告诉你"我不确定了"，而是自信满满地给你写一段跟前面矛盾的代码。你如果不仔细 review，根本发现不了。

Debug 是它的死穴

如果写新代码是 80 分，那 debug 可能只有 40 分。

遇到那种报错信息模糊的问题——比如并发导致的偶发 bug、内存泄漏、时序相关的状态错误——Claude Code 基本上是在瞎猜。它会反复修改同一块代码，每次都信誓旦旦说"这次应该修好了"，然后跑测试还是挂。最夸张的一次，它对着一个 race condition 改了七轮，每一轮的思路都不一样，但没有一轮是对的。

最后还是我自己打断它，花二十分钟读了日志，定位到问题在另一个模块。那种感觉很挫败——你花了一个小时等它试错，其实自己上手五分钟就能找到方向。

任务碎片化的问题

Claude Code 做小事很强，但它很难把一系列小任务有机地串联起来。比如你要重构一个模块，这需要先理解现有代码、设计新结构、逐步迁移、保证测试通过。每一步它都能做，但合在一起就经常出问题——因为它没有全局视角，每一步都是局部最优。

它更像一个执行力很强但没有大局观的初级工程师。你说一步它做一步，但它不会主动想"哎这个改动会不会影响昨天改的那个模块"。

我现在怎么用它

放弃用它写代码，不等于不用它。我现在把它当一个非常快的助手，而不是一个可以撒手不管的代理人。

具体来说：

  • 项目起步阶段：让它搭骨架、写样板代码、配环境。这个阶段需求简单、边界清晰，是它最舒服的场景。
  • 写一次性脚本：数据处理、格式转换、批量操作这类写完就扔的代码，交给它效率极高。
  • 补测试和文档：这两样东西人类不爱写，AI 不在乎，简直是天作之合。
  • 代码审查助手：把一段代码扔给它，让它找 bug、提建议。不是让它改，是让它看。它的视角跟人不一样，有时候能发现你忽略的问题。

但涉及到核心逻辑、架构设计、复杂 bug 的调试——这些我自己来。

说句实在话，这个分工反而让我的效率提高了。以前让 Claude Code 全权负责，我花大量时间在检查和修复它的问题；现在我只让它干它擅长的事，剩下的自己掌控，整体反而更快更稳。

什么人该用，什么人不该用

如果你是一个有经验的工程师，清楚自己在做什么，只是想省掉那些无聊的重复劳动——Claude Code 值得一试。你知道什么该交给它，什么该自己做，它能实实在在地帮你提效。

如果你是一个刚入门的新手，指望用它来"代替写代码"——我劝你慎重。不是因为它写的代码不能跑，而是因为你看不出它写错了什么。当你没有能力判断它的输出质量时，依赖它只会让你走更多弯路。

还有一类人：独立开发者或者小团队，项目不大但什么都得自己干。Claude Code 对你们来说可能特别值——它能帮你把那些你没精力做但不得不做的事快速搞定，让你把精力集中在真正需要思考的部分。

工具好不好，不在于它多强大，在于你能不能把它放在合适的位置。Claude Code 不是替代你的编码能力，是放大你愿意花心思的那部分。

半年前我以为它是自动写代码的未来。现在我觉得，它更像一个跑得很快但你得牵着绳的狗——方向得你来定，否则它跑得越快，离目的地越远。
