NAS跑AI大模型指南：Ollama部署DeepSeek/Qwen/Llama3，NAS变身本地ChatGPT

2026-04-15 · 数码硬件/NAS存储 · 🔖 NAS, AI大模型, Ollama, DeepSeek, 本地AI, 本地LLM, ChatGPT, 私有AI

DeepSeek火了一整年，想在本地跑个AI助手，但电脑带不动？

你的NAS完全可以。只要内存够（8GB以上），就能在NAS上跑DeepSeek、Qwen、LLaMA等大模型，数据不离开局域网，隐私安全，响应飞快。

一、为什么在NAS上跑AI

云端API（OpenAI、Claude、DeepSeek官方API）虽然方便，但有几个问题：

  • 隐私风险：你的对话数据存在别人服务器上
  • 网络依赖：断网就用不了
  • 费用累积：用多了账单吓人
  • 延迟波动：高峰期响应慢

NAS跑本地AI的优势：

  • 数据完全私有：所有对话存在本地，家人照片、文档、工作内容不外传
  • 7x24在线：NAS本来就是一直开着的，随时响应
  • 零使用成本：一次部署，无限使用
  • 局域网延迟：毫秒级响应，体验流畅

二、Ollama是什么

本地跑大模型，绕不开Ollama这个工具。

Ollama是一个开源的本地大模型运行框架，核心特点：

  • 一键部署：一条命令下载并运行模型，不需要配置环境
  • 模型库丰富：DeepSeek、Qwen、LLaMA、Mistral、Gemma等主流模型一键获取
  • API兼容：提供OpenAI兼容API，现有应用可以直接接入
  • 跨平台：支持macOS/Linux/Windows，NAS上用Docker运行
  • 资源优化：智能管理CPU/GPU资源，NAS也能跑得动

三、群晖Docker部署Ollama

方案一：Docker直接部署（推荐）

Step 1：SSH连接到群晖

# 在终端输入
ssh admin@192.168.1.100
# 输入密码后切换到root
sudo -i

Step 2：创建文件夹

mkdir -p /volume1/docker/ollama

Step 3：创建docker-compose.yml

cat > /volume1/docker/ollama/docker-compose.yml << 'EOF'
version: '3.8'
services:
  ollama:
    image: ollama/ollama:latest
    container_name: ollama
    restart: unless-stopped
    ports:
      - "11434:11434"
    volumes:
      - ./data:/root/.ollama
    environment:
      - OLLAMA_HOST=0.0.0.0
    deploy:
      resources:
        limits:
          memory: 8G
EOF

Step 4：启动

cd /volume1/docker/ollama
docker-compose up -d

Step 5：下载模型

# 进入容器
docker exec -it ollama ollama run deepseek-r1:1.5b

# 其他常用模型
docker exec -it ollama ollama run qwen2.5:7b
docker exec -it ollama ollama run llama3.2:3b
docker exec -it ollama ollama run mistral:7b

方案二：Open WebUI（图形界面）

纯命令行的ollama不好用？装个Web界面——Open WebUI，体验和ChatGPT几乎一样。

Step 1：部署Open WebUI

docker run -d \
  --name open-webui \
  --network host \
  -v ./open-webui/data:/app/backend/data \
  -e OLLAMA_BASE_URL=http://127.0.0.1:11434 \
  --restart unless-stopped \
  ghcr.io/open-webui/open-webui:main

Step 2：访问

浏览器打开http://NAS的IP:8080，注册账号后就能用了。

💡 推荐组合：Ollama（后端）+ Open WebUI（前端），这是目前最流行的本地AI方案，体验接近官方ChatGPT。

四、威联通Container Station部署

Step 1：安装Container Station

App Center搜索并安装Container Station。

Step 2：创建容器

  1. 打开Container Station，点击创建
  2. 镜像名称填：ollama/ollama:latest
  3. 网络模式选择桥接（Bridge），端口映射11434:11434
  4. 存储空间挂载：新建一个ollama文件夹，挂载到/root/.ollama
  5. 创建并启动

Step 3：下载模型

# SSH到威联通
ssh admin@NAS的IP

# 进入容器
docker exec -it ollama ollama run deepseek-r1:1.5b

五、推荐模型

不是所有模型都能在NAS上跑。参数越大，能力越强，但对硬件要求也越高。

模型选择指南

模型 参数量 推荐内存 适用场景 推荐度
DeepSeek-R1 1.5B 4GB 日常问答、快速响应 ⭐⭐⭐⭐⭐
DeepSeek-R1 7B 8GB 文档处理、代码编写 ⭐⭐⭐⭐
Qwen2.5 3B 6GB 中文对话、轻量任务 ⭐⭐⭐⭐
Qwen2.5 7B 8GB 复杂推理、中文创作 ⭐⭐⭐⭐⭐
LLaMA3.2 3B 6GB 英文为主的任务 ⭐⭐⭐
Mistral 7B 8GB 通用任务、代码 ⭐⭐⭐
Phi-3.5 3.8B 6GB 轻量级、快速响应 ⭐⭐⭐⭐
⚠️ 内存是瓶颈：NAS跑大模型，内存决定上限。建议至少8GB，16GB体验更好。如果只有4GB，只能跑1.5B参数模型。

我的推荐

  • 入门首选：DeepSeek-R1 1.5B，体积小、速度快、中文好
  • 主力使用：DeepSeek-R1 7B或Qwen2.5 7B，能力均衡
  • NAS内存16GB+：可以跑14B参数的模型，效果接近GPT-3.5

六、进阶：本地知识库（RAG）

光跑模型还不够？接上你的文档，让AI读你的笔记、合同、代码——这就是RAG（检索增强生成）。

AnythingLLM（最简单）

  1. 部署AnythingLLM容器
  2. 连接Ollama作为模型
  3. 上传PDF、Word、Markdown文档
  4. AI就能回答关于你文档的问题
docker run -d \
  --name anythingllm \
  -p 3001:3001 \
  -v anythingllm:/app/storage \
  -e OLLAMA_BASE_URL=http://NAS的IP:11434 \
  mintplexlabs/anythingllm:latest

配合Dify（更强大）

Dify是一个AI应用编排平台，可以：

  • 连接Ollama模型
  • 构建知识库
  • 创建AI工作流
  • 开发智能客服、助手类应用

七、API调用（开发者向け）

Ollama提供OpenAI兼容的API，可以直接替换你的AI应用后端地址。

API地址

# Ollama API地址
http://NAS的IP:11434

# 模型列表
curl http://NAS的IP:11434/api/tags

# 对话请求（OpenAI兼容格式）
curl http://NAS的IP:11434/v1/chat/completions \
  -H "Content-Type: application/json" \
  -d '{
    "model": "deepseek-r1:1.5b",
    "messages": [{"role": "user", "content": "你好"}]
  }'

支持OpenAI SDK

如果你的代码用的是OpenAI SDK，只需改一行配置：

# Python OpenAI SDK示例
from openai import OpenAI

client = OpenAI(
    api_key="ollama",  # 任意字符串
    base_url="http://NAS的IP:11434/v1"  # Ollama地址
)

response = client.chat.completions.create(
    model="deepseek-r1:1.5b",
    messages=[{"role": "user", "content": "用Python写一个快速排序"}]
)
print(response.choices[0].message.content)

八、硬件要求

模型参数 最低内存 推荐内存 CPU要求 推理速度参考
1.5B 4GB 6GB 任意x86 20-50 tokens/s
3B 6GB 8GB 四核以上 10-30 tokens/s
7B 8GB 16GB 四核以上 5-15 tokens/s
14B 16GB 32GB 六核以上 2-8 tokens/s
💡 没有GPU也能跑：NAS的CPU跑纯CPU推理是可行的，只是比GPU慢一些。日常使用足够，等人不耐烦就不行。

十、常见问题

问题 原因 解决
模型下载很慢 默认从境外下载 挂梯子，或使用镜像站（如魔搭）
内存不足报错 模型太大 换更小的模型，或关闭其他容器释放内存
响应很慢 CPU性能不足 这是正常现象，没有GPU就是这样
Open WebUI连不上Ollama 网络模式不对 使用host网络模式，或检查端口映射
模型回答乱码 编码问题 在OLLAMA_HOST环境变量后加LANG=C.UTF-8
模型选择列表为空 还没下载模型 先执行docker exec -it ollama ollama run deepseek-r1:1.5b

总结

NAS跑AI大模型，已经非常成熟了：

  1. Ollama：一条命令跑模型，最简单
  2. Open WebUI：图形界面，体验接近ChatGPT
  3. AnythingLLM：接知识库，读你的文档
  4. 内存决定上限：8GB跑7B，16GB跑14B
  5. 纯CPU可用：没有GPU也能跑，就是慢点
别让NAS只存文件。让它成为你的私有AI大脑，24小时在线，永不疲倦。
