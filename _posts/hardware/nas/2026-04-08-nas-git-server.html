NAS搭建Git服务器：搭建私有代码仓库

2026-04-08 · 数码硬件/NAS存储 · 🔖 NAS, Git, Gitea, GitLab, 代码仓库, 版本控制

想拥有自己的代码托管平台？不想把代码放在GitHub上？NAS可以帮你搭建私有Git服务器！

本文详细讲解在NAS上搭建Git服务器的各种方案，从简单的命令行到功能完善的Web界面。

二、Gitea（推荐方案）

Gitea是一款轻量级的开源Git服务，功能完善且资源占用低，非常适合NAS使用。

Gitea特点

  • 开源免费，社区活跃
  • 界面类似GitHub，易于上手
  • 支持Issue、Pull Request、Wiki
  • 支持组织、团队权限管理
  • 支持Webhook、CI/CD集成
  • 资源占用低（内存仅需几百MB）

Docker安装Gitea

步骤1：创建目录

mkdir -p /volume1/docker/gitea/data
mkdir -p /volume1/docker/gitea/config

步骤2：运行容器

docker run -d \
  --name=gitea \
  --restart=always \
  -p 3000:3000 \
  -p 222:22 \
  -v /volume1/docker/gitea/data:/data \
  -v /volume1/docker/gitea/config:/data/gitea/conf \
  -e USER_UID=1000 \
  -e USER_GID=1000 \
  gitea/gitea:latest

参数说明：

  • -p 3000:3000：Web界面端口
  • -p 222:22：SSH端口（避免与NAS SSH冲突）
  • -v：数据持久化映射

初始化配置

步骤1：访问安装页面

  1. 浏览器访问 http://NAS_IP:3000
  2. 进入安装配置页面

步骤2：数据库设置

  • 数据库类型：SQLite（简单）或MySQL/PostgreSQL（推荐长期使用）
  • SQLite适合个人使用，无需额外配置
  • MySQL/PostgreSQL适合团队使用，性能更好

步骤3：基本设置

配置项 建议值
站点标题 你的Git服务器名称
仓库根目录 /data/git/repositories
SSH服务域名 NAS的IP或域名
SSH服务端口 222（与NAS SSH区分）
基础URL http://NAS_IP:3000/

步骤4：创建管理员账号

  1. 设置管理员用户名
  2. 设置邮箱和密码
  3. 点击"安装Gitea"

使用Gitea

创建仓库

  1. 登录Gitea
  2. 点击右上角"+" → "新建仓库"
  3. 填写仓库名称和描述
  4. 选择公开或私有
  5. 点击"创建仓库"

克隆仓库

# HTTP方式
git clone http://NAS_IP:3000/用户名/仓库名.git

# SSH方式（需配置SSH密钥）
git clone ssh://git@NAS_IP:222/用户名/仓库名.git

配置SSH密钥

  1. 本地生成SSH密钥：ssh-keygen -t rsa -C "your_email@example.com"
  2. 复制公钥内容：cat ~/.ssh/id_rsa.pub
  3. Gitea → 设置 → SSH/GPG密钥 → 添加密钥

Gitea高级功能

Issue跟踪

  • 创建Issue记录bug或任务
  • 分配责任人、设置标签、里程碑
  • 支持Markdown格式

Pull Request

  • 代码审查流程
  • 分支合并前审核
  • 评论和讨论

Webhook

  • 推送事件触发外部服务
  • 集成CI/CD工具
  • 自动部署

组织管理

  • 创建组织管理多个仓库
  • 团队成员权限控制
  • 组织级别的设置

三、GitLab CE

GitLab是功能最全面的Git服务，但资源占用较高，适合配置较好的NAS。

系统要求

  • CPU：4核以上
  • 内存：4GB以上（推荐8GB）
  • 存储：根据代码量决定
⚠️ 注意：GitLab启动较慢，首次启动可能需要几分钟。内存小于4GB的NAS不建议安装。

Docker安装GitLab

步骤1：创建目录

mkdir -p /volume1/docker/gitlab/config
mkdir -p /volume1/docker/gitlab/logs
mkdir -p /volume1/docker/gitlab/data

步骤2：运行容器

docker run -d \
  --name gitlab \
  --hostname gitlab \
  --restart always \
  -p 8080:80 \
  -p 8443:443 \
  -p 2222:22 \
  -v /volume1/docker/gitlab/config:/etc/gitlab \
  -v /volume1/docker/gitlab/logs:/var/log/gitlab \
  -v /volume1/docker/gitlab/data:/var/opt/gitlab \
  gitlab/gitlab-ce:latest

步骤3：等待启动

  1. 首次启动需要几分钟初始化
  2. 查看日志：docker logs -f gitlab
  3. 看到"gitlab Reconfigured!"表示启动完成

步骤4：初始配置

  1. 访问 http://NAS_IP:8080
  2. 设置管理员密码（root账号）
  3. 使用root登录

GitLab特点

  • 功能最全面（CI/CD、Registry、Pages等）
  • 企业级权限管理
  • 内置CI/CD流水线
  • 容器镜像仓库
  • Wiki、Issue、Merge Request

四、群晖Git Server套件

群晖官方提供的Git服务，配置简单但功能较基础。

安装步骤

步骤1：安装套件

  1. 打开【套件中心】
  2. 搜索【Git Server】
  3. 点击安装

步骤2：创建共享文件夹

  1. 打开【控制面板】→【共享文件夹】
  2. 创建用于存放Git仓库的文件夹（如git_repos）

步骤3：创建Git用户

  1. 【控制面板】→【用户与群组】→【用户账号】
  2. 点击【新增】
  3. 创建专用Git用户（如git）
  4. 赋予git_repos文件夹的读写权限

步骤4：配置Git Server

  1. 打开【Git Server】套件
  2. 勾选允许访问的Git用户
  3. 保存设置

步骤5：启用SSH

  1. 【控制面板】→【终端机和SNMP】
  2. 勾选【启用SSH服务】
  3. 保存

步骤6：启用家目录

  1. 【控制面板】→【用户与群组】→【高级设置】
  2. 启用【家目录服务】

创建Git仓库

通过SSH连接到NAS创建仓库：

# SSH连接到NAS（使用管理员账号）
ssh admin@NAS_IP

# 进入Git仓库目录
cd /volume1/git_repos

# 创建项目目录
mkdir myproject.git
cd myproject.git

# 初始化裸仓库
git init --bare

# 设置权限（让git用户可以访问）
sudo chown -R git:users /volume1/git_repos/myproject.git

使用仓库

# 克隆仓库
git clone ssh://git@NAS_IP/volume1/git_repos/myproject.git

# 或使用HTTP（需配置Web服务器）
git clone http://NAS_IP/git/myproject.git

五、裸仓库（极简方案）

最简单的Git服务器，不需要任何Web界面，纯命令行操作。

创建裸仓库

# 在NAS上创建仓库目录
mkdir -p /volume1/git/repo.git
cd /volume1/git/repo.git

# 初始化裸仓库
git init --bare

本地使用

# 添加远程仓库
git remote add origin ssh://user@NAS_IP/volume1/git/repo.git

# 推送代码
git push -u origin master
💡 提示：裸仓库适合个人使用或作为备份，没有Web界面，需要通过SSH操作。

六、方案选择建议

个人开发者

推荐：Gitea

  • 资源占用低
  • 功能完善
  • 界面友好
  • 易于维护

小团队（2-10人）

推荐：Gitea或GitLab CE

  • 需要CI/CD：选GitLab
  • 追求轻量：选Gitea
  • 权限管理：两者都支持

仅做备份

推荐：裸仓库或群晖Git Server

  • 配置最简单
  • 资源占用最低
  • 满足基本需求

七、安全建议

SSH密钥认证

  • 禁用密码登录，使用SSH密钥
  • 定期更换密钥
  • 不同设备使用不同密钥

访问控制

  • 私有仓库默认不公开
  • 团队成员按需分配权限
  • 定期审查访问权限

备份策略

  • Git仓库定期备份到其他设备
  • 使用Hyper Backup或HBS 3
  • 异地备份到云端

HTTPS配置

  • 使用反向代理配置HTTPS
  • 申请Let's Encrypt证书
  • 强制HTTPS访问

八、常见问题

Q：Gitea和GitLab怎么选？

A：NAS配置好选GitLab，追求轻量选Gitea。GitLab功能全面但吃资源，Gitea够用且省资源。

Q：外网怎么访问？

A：配置DDNS+端口转发，或使用内网穿透。建议配合反向代理使用HTTPS。

Q：代码会丢失吗？

A：Git是分布式版本控制，本地和远程都有完整历史。但建议定期备份NAS数据。

Q：多人协作冲突怎么办？

A：使用分支开发，Pull Request合并。冲突时手动解决，Git会提示冲突位置。

Q：可以替代GitHub吗？

A：功能上可以，但GitHub的生态（Actions、Issues社区等）更丰富。私有项目用NAS，开源项目用GitHub。

九、CI/CD集成

在NAS上搭建Git服务器后，可以配合CI/CD工具实现自动化：

Gitea + Drone CI

# 安装Drone
docker run -d \
  --name=drone \
  -p 8081:80 \
  -v /volume1/docker/drone:/data \
  -e DRONE_GITEA_SERVER=http://NAS_IP:3000 \
  -e DRONE_RPC_SECRET=secret \
  drone/drone:latest

GitLab CI

GitLab内置CI/CD，在仓库根目录创建.gitlab-ci.yml：

stages:
  - build
  - test
  - deploy

build:
  stage: build
  script:
    - echo "Building..."
    - make build

test:
  stage: test
  script:
    - echo "Testing..."
    - make test

deploy:
  stage: deploy
  script:
    - echo "Deploying..."
    - make deploy

总结

在NAS上搭建Git服务器，让代码管理更自主：

选择适合自己需求的方案，开始搭建私有代码仓库吧！
