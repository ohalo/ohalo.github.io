NAS音乐中心搭建指南：打造私人音乐流媒体服务

2026-04-08 · 数码硬件/NAS存储 · 🔖 NAS, 音乐, 流媒体, Navidrome, Audio Station, 音乐服务器

音乐APP会员到期、歌单变灰、喜欢的歌被下架...

与其依赖各种音乐平台，不如在NAS上搭建自己的音乐中心。本文教你如何把NAS变成私人音乐流媒体服务器。

一、为什么要自建音乐中心

流媒体音乐的痛点

  • 会员到期，下载的歌无法播放
  • 版权到期，歌单里的歌突然变灰
  • 音质压缩，无损音乐额外收费
  • 广告干扰，体验越来越差
  • 个性化推荐，越听越窄

NAS音乐中心的优势

优势 说明
永久拥有 音乐文件保存在本地，不用担心下架
无损音质 支持FLAC、WAV等无损格式
无广告 纯粹的听歌体验
多设备同步 手机、电脑、车机都能播放
智能管理 自动整理、歌词显示、封面匹配

二、主流方案对比

方案 适用NAS 优点 缺点
群晖 Audio Station 群晖 官方套件、简单易用 功能较简单、可能停止维护
Navidrome 全平台 开源免费、资源占用低、功能丰富 需Docker或第三方套件
Jellyfin 全平台 音视一体、界面美观 音乐功能不如专业音乐服务器
Airsonic 全平台 老牌方案、客户端丰富 界面较旧、更新慢
💡 推荐：群晖用户首选Audio Station（简单）或Navidrome（功能强）；其他NAS用户推荐Navidrome。

三、群晖Audio Station

安装Audio Station

  1. 打开【套件中心】
  2. 搜索【Audio Station】
  3. 点击【安装套件】

导入音乐

  1. 将音乐文件放入NAS的【music】共享文件夹
  2. Audio Station会自动扫描并索引
  3. 支持格式：MP3、AAC、FLAC、WAV、OGG、WMA等

界面功能

  • 按文件夹浏览：保持原有文件夹结构
  • 按艺术家/专辑/流派：自动分类整理
  • 播放列表：创建自定义歌单
  • 广播：收听网络电台

显示封面和歌词

Audio Station会自动从互联网获取封面和歌词，但也可以手动整理：

  • 封面图片命名为cover.jpg放在专辑文件夹
  • 歌词文件命名为与歌曲同名，后缀.lrc

手机APP

  • iOS：【DS audio】
  • Android：【DS audio】
  • 支持下载到本地离线播放

车机支持

  • 支持CarPlay
  • 支持Android Auto
  • 开车时也能听NAS里的歌
⚠️ 注意：群晖官方已暗示Audio Station可能停止维护，建议新用户考虑Navidrome。

四、Navidrome（推荐）

什么是Navidrome

Navidrome是一款开源的自托管音乐服务器，特点：

  • 资源占用极低（树莓派都能跑）
  • 支持几乎所有音频格式
  • 兼容Subsonic API（客户端丰富）
  • 支持即时转码
  • 支持Opus编码
  • 多用户支持
  • 中文界面

群晖安装Navidrome（套件方式）

步骤1：添加套件源

  1. 打开【套件中心】
  2. 点击【设置】→【套件来源】
  3. 点击【新增】
  4. 名称：矿神源
  5. 位置：https://spk7.imnks.com/
  6. 点击确定

步骤2：安装Navidrome

  1. 在套件中心左侧选择【社群】
  2. 搜索【Navidrome】
  3. 点击【安装套件】
  4. 选择安装位置

步骤3：设置权限

  1. 打开【控制面板】→【共享文件夹】
  2. 选择存放音乐的文件夹
  3. 点击【编辑】→【权限】
  4. 选择【系统内部用户账号】
  5. 找到【sc-navidrome】
  6. 勾选【读取】权限

步骤4：配置Navidrome

  1. 安装完成后点击【打开】
  2. 设置管理员用户名和密码
  3. 进入设置界面，选择中文
  4. 设置音乐库路径（默认是/music）

Docker安装Navidrome（通用方式）

适合所有支持Docker的NAS：

docker run -d \
  --name navidrome \
  -p 4533:4533 \
  -v /path/to/music:/music \
  -v /path/to/data:/data \
  --restart unless-stopped \
  deluan/navidrome:latest

参数说明：

  • -p 4533:4533：映射端口
  • -v /path/to/music:/music：音乐文件夹映射
  • -v /path/to/data:/data：数据文件夹映射

Navidrome使用

Web界面

  • 浏览器访问 http://NAS_IP:4533
  • 登录后即可浏览和播放
  • 界面现代美观，支持暗黑模式

自动扫描

  • Navidrome会自动监控音乐文件夹变化
  • 新增音乐自动导入
  • 元数据变更自动更新

第三方客户端

Navidrome兼容Subsonic API，可使用众多第三方客户端：

平台 推荐客户端 特点
iOS play:Sub、Substreamer 功能完善、界面美观
Android DSub、Ultrasonic 开源免费、支持下载
Windows/Mac Sublime Music、Sonixd 桌面级体验
Web Navidrome自带 无需安装
💡 推荐：Android用户推荐【音流】APP，专为Navidrome优化，体验非常好。

│   └── Ed Sheeran/
├── 日语/

元数据标签

确保音乐文件有完整的ID3标签：

  • 标题（Title）
  • 艺术家（Artist）
  • 专辑（Album）
  • 年份（Year）
  • 流派（Genre）
  • 音轨号（Track Number）

封面和歌词

自动获取工具

  • MusicTag（Windows）：批量获取封面和歌词
  • music_tag_web（Docker）：Web版标签编辑器
  • Beets（命令行）：自动整理和标签补全

手动整理

  • 封面：专辑文件夹下放cover.jpg或folder.jpg
  • 歌词：与歌曲同名，后缀.lrc

六、音质与格式

常见音频格式

格式 压缩 音质 文件大小 适用场景
MP3 有损 中等 小 通用兼容
AAC 有损 较好 小 苹果设备
FLAC 无损 优秀 大 收藏首选
WAV 无损 优秀 很大 专业用途
DSD 无损 顶级 极大 发烧友

推荐收藏格式

  • 日常听歌：FLAC（音质好、兼容性好）
  • 移动设备：AAC 256kbps（省空间）
  • 发烧收藏：DSD或Hi-Res FLAC

七、远程访问

内网穿透方案

出门在外也想听NAS里的歌？需要配置远程访问：

方案 适用场景 难度
QuickConnect（群晖） 群晖用户 简单
myQNAPcloud（威联通） 威联通用户 简单
DDNS+端口映射 有公网IP 中等

💡 建议：优先使用官方提供的远程访问服务（如QuickConnect），简单稳定。

八、常见问题

Q：Navidrome识别不到音乐？

A：检查文件夹权限，确保navidrome用户有读取权限；检查音乐文件格式是否支持。

Q：封面和歌词不显示？

A：使用MusicTag等工具补全元数据；确保封面图片命名为cover.jpg。

Q：手机播放卡顿？

A：开启转码功能，将FLAC转为AAC或MP3，降低带宽需求。

Q：如何备份音乐库？

A：音乐文件本身就是备份，建议定期将NAS音乐文件夹同步到另一设备或云端。

Q：支持多少首歌曲？

A：Navidrome资源占用很低，轻松支持几万首歌曲。

九、进阶玩法

智能歌单

  • 按心情分类（工作、运动、睡前）
  • 按场景分类（开车、学习、聚会）
  • 自动随机播放

多用户管理

  • 为家人创建独立账号
  • 设置不同的访问权限
  • 各自的歌单互不影响

智能音箱对接

  • 部分智能音箱支持Subsonic API
  • 可用NAS音乐替换在线音乐源

总结

自建NAS音乐中心，告别音乐APP的各种限制：

  • 群晖用户：Audio Station简单易用，Navidrome功能更强
  • 其他NAS用户：Navidrome是最佳选择
  • 音乐整理：良好的文件夹结构和完整的元数据很重要
  • 远程访问：配置好就能随时随地听歌

推荐配置：

  • 音乐存储：FLAC格式，按艺术家/专辑整理
  • 服务器：Navidrome（Docker部署）
  • 客户端：音流（Android）或play:Sub（iOS）
  • 标签整理：MusicTag
音乐是生活的调味剂，自建音乐中心让听歌回归纯粹。
