NAS远程访问进阶：Tailscale零配置VPN、WireGuard自建、frp内网穿透方案对比

2026-04-15 · 数码硬件/NAS存储 · 🔖 NAS, 远程访问, Tailscale, WireGuard, VPN, 内网穿透, frp, Cloudflare Tunnel

在外面想访问家里的NAS，但没公网IP？QuickConnect太慢？

这篇文章介绍三种进阶远程访问方案：零配置的Tailscale、自建WireGuard VPN、灵活的frp内网穿透。总有一款适合你。

二、Tailscale（强烈推荐）

Tailscale是目前最推荐的远程访问方案。基于WireGuard协议，但完全零配置——不需要懂网络知识，装完就能用。

核心特点

  • 零配置：不需要改路由器、不需要端口映射、不需要公网IP
  • P2P直连：设备间直连，延迟低，速度等于带宽上限
  • 多平台支持：iOS、Android、Windows、Mac、Linux、NAS
  • 免费额度：个人用户最多20台设备，完全够用
  • 免费子域名+HTTPS：可以白嫖SSL证书

群晖安装步骤

方法一：套件中心（推荐DSM 7.x）

  1. 打开Tailscale官网（tailscale.com），用微软/谷歌/GitHub账号登录
  2. 进入管理后台 → Settings → Keys → Generate auth key，勾选Reusable后复制密钥
  3. 在群晖套件中心搜索并安装Tailscale（或从GitHub下载spk包手动安装）
  4. 打开Tailscale，输入auth key完成连接
  5. 管理后台看到NAS设备在线，获得一个100.x.x.x的虚拟IP

方法二：Docker（所有NAS通用）

docker run -d \
  --name tailscale \
  --network host \
  --privileged \
  -v /volume1/docker/tailscale:/var/lib/tailscale \
  -e TS_AUTHKEY=your-auth-key-here \
  ghcr.io/tailscale/tailscale:latest \
  tailscaled

启动后进入容器执行连接：

docker exec -it tailscale tailscale up --accept-routes

威联通安装步骤

  1. 在QNAP Club应用商店搜索Tailscale，安装
  2. 打开Tailscale，用同一账号登录
  3. 获得虚拟IP，在管理后台确认设备在线

客户端使用

在你需要远程访问NAS的设备上（手机、电脑）安装Tailscale客户端，用同一账号登录。

之后在外面想访问NAS时：

  • 打开Tailscale，确认已连接
  • 在文件管理器/浏览器输入NAS的虚拟IP（如100.92.18.50）
  • 就像在局域网一样访问NAS所有服务

白嫖HTTPS证书（进阶）

Tailscale提供免费子域名 + HTTPS证书：

  1. 管理后台 → DNS → 添加免费域名（如my-nas.tailABCD.ts.net）
  2. SSH到NAS，执行：sudo tailscale cert my-nas.tailABCD.ts.net
  3. 获得SSL证书，配置到Web服务
  4. 用域名访问，群晖DSM也可以开启HTTPS
💡 Tailscale免费版限制：最多20台设备，不支持自定义子网路由，不支持Exit Node（全局代理）。普通用户完全够用。

三、WireGuard自建VPN

如果你有一台境外VPS，可以自建WireGuard VPN，速度更快，且可以科学上网。

原理

WireGuard是一个轻量级VPN协议，比OpenVPN快很多。你的设备和VPS之间建立加密隧道，所有流量通过VPS转发。

VPS服务器配置

# 安装WireGuard
apt update && apt install -y wireguard

# 生成密钥
cd /etc/wireguard
wg genkey | tee server_private.key | wg pubkey > server_public.key
wg genkey | tee client_private.key | wg pubkey > client_public.key

# 配置服务器
cat > wg0.conf << 'EOF'
[Interface]
PrivateKey = $(cat server_private.key)
Address = 10.0.0.1/24
ListenPort = 51820
PostUp = iptables -A FORWARD -i wg0 -j ACCEPT; iptables -t nat -A POSTROUTING -o eth0 -j MASQUERADE
PostDown = iptables -D FORWARD -i wg0 -j ACCEPT; iptables -t nat -D POSTROUTING -o eth0 -j MASQUERADE

[Peer]
PublicKey = $(cat client_public.key)
AllowedIPs = 10.0.0.2/32
EOF

NAS客户端配置

# 群晖Docker运行WireGuard
docker run -d \
  --name wireguard \
  --network host \
  --privileged \
  -v /volume1/docker/wireguard:/config \
  -e PEERS=1 \
  -e SERVERURL=你的VPS的IP \
  -e SERVERPORT=51820 \
  -e INTERNAL_SUBNET=10.0.0.0/24 \
  -e PEERDNS=8.8.8.8 \
  -e ALLOWEDIPS=0.0.0.0/0 \
  --restart unless-stopped \
  linuxserver/wireguard

启用IP转发（服务器）

echo "net.ipv4.ip_forward = 1" >> /etc/sysctl.conf
sysctl -p
⚠️ 注意：WireGuard需要一台始终在线的VPS。VPS建议选离你近的节点，速度更好。阿里云国际版、CloudCone、Vultr都是常见选择。

四、frp内网穿透

frp（Fast Reverse Proxy）是一个开源的内网穿透工具，需要一台有公网IP的服务器（中继服务器）。

适用场景

  • 需要暴露固定端口给外部访问
  • 有多台服务需要同时暴露
  • 有可用的中继服务器（自己的VPS或免费frp服务）

群晖Docker部署（frpc客户端）

Step 1：准备配置文件

mkdir -p /volume1/docker/frpc
cat > /volume1/docker/frpc/frpc.ini << 'EOF'
[common]
server_addr = 你的中继服务器IP
server_port = 7000
token = 你的密码

[nas_dsm]
type = tcp
local_ip = 127.0.0.1
local_port = 5000
remote_port = 15500

[nas_file]
type = tcp
local_ip = 127.0.0.1
local_port = 5001
remote_port = 15501
EOF

Step 2：启动容器

docker run -d \
  --name frpc \
  --restart unless-stopped \
  -v /volume1/docker/frpc/frpc.ini:/etc/frp/frpc.ini \
  snowdreamtech/frpc

服务器部署（frps服务端）

# VPS上安装frps
wget https://github.com/fatedier/frp/releases/download/v0.58.0/frp_0.58.0_linux_amd64.tar.gz
tar -xzf frp_0.58.0_linux_amd64.tar.gz
cd frp_0.58.0_linux_amd64

# 配置
cat > frps.ini << 'EOF'
[common]
bind_port = 7000
token = 你的密码
dashboard_port = 7500
dashboard_user = admin
dashboard_pwd = 你的后台密码
EOF

# 启动
./frps -c frps.ini

访问方式

在外网访问中继服务器IP:15500即可连接到群晖DSM。

💡 免费frp服务：网上有一些公益frp服务（如waiwang.men），可以直接用他们的服务器，省去自己搭建。但稳定性和速度不如自建。

五、Cloudflare Tunnel

Cloudflare Tunnel（原frangipani）是目前最稳定的免费方案。不需要任何配置，流量经过Cloudflare全球CDN，速度快且稳定。

部署步骤

# 群晖Docker运行cloudflare tunnel
docker run -d \
  --name cloudflare-tunnel \
  --network host \
  -v /volume1/docker/cloudflare-tunnel:/etc/cloudflared \
  cloudflare/cloudflared:latest \
  tunnel run --token 你的token

在Cloudflare Zero Trust后台创建Tunnel，获取token后填入即可。

优势

  • 免费
  • 自动HTTPS
  • Cloudflare全球CDN加速
  • 支持子域名绑定
  • 稳定，不掉线

七、进阶技巧

1. 多服务一键访问

用Tailscale + 一个固定域名，可以通过一个入口访问所有服务：

# 在NAS上用Nginx反向代理
server {
    listen 80;
    server_name nas.tailABCD.ts.net;
    
    location / {
        proxy_pass http://127.0.0.1:5000;
    }
    
    location /files/ {
        proxy_pass http://127.0.0.1:5001;
    }
}

2. 手机APP访问

DS file、DS photo、DS video等APP都支持自定义服务器地址：

  1. 打开APP → 设置 → 服务器
  2. 添加服务器，输入Tailscale虚拟IP或域名
  3. 勾选"使用HTTPS"（如果有SSL证书）
  4. 连接成功

3. 开机自启

确保Tailscale在NAS启动后自动连接：

  • Docker方式：容器设置中勾选"重启后自动启动"
  • 套件方式：Tailscale套件设置中开启开机自启

八、常见问题

问题 解答
Tailscale连不上 检查客户端是否登录同一账号，NAT类型是否限制了P2P，切换到DERP中继模式
速度很慢 国内直连Tailscale中继服务器慢，可尝试换成 nearer DERP节点
frp中继慢 中继服务器位置决定速度，换一台近的VPS
端口被封了 WireGuard用UDP 51820，部分运营商会封锁，换端口或用Tailscale
Tailscale免费版限制 20台设备上限，够用；不支持Exit Node（需要付费版）
手机APP访问Tailscale IP 在DS file等APP中手动添加服务器，地址填Tailscale虚拟IP
多个设备都要连NAS 所有设备都装Tailscale，用同一账号，NAS只需一个虚拟IP

总结

  1. 日常远程访问：Tailscale是最佳选择，零配置，够快，完全免费
  2. 需要HTTPS + 自定义域名：Cloudflare Tunnel，稳定可靠
  3. 有VPS，追求极速：WireGuard自建，速度等于VPS带宽
  4. 多服务暴露：frp最灵活，需要中继服务器
  5. 懒人方案：蒲公英/花生壳，开箱即用但有限速
远程访问NAS，核心就一句话：Tailscale治百病。试过就知道，QuickConnect是什么垃圾。
