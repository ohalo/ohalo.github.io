10个技巧，通关Claude Code

2026-04-07

说一个很扎心的事实：大多数人在用 Claude Code 的方式，从根上就错了。

你以为调教 AI 编程就是扔一句 prompt、等着它给你写代码？Claude Code 团队自己公开的 10 条私藏技巧，看完你会发现自己之前全在"糙用"。

这 10 条技巧来自 Claude Code 作者 Boris 在 Twitter 上的热门帖子，每一个都经过实战验证。每一位用 AI 编程的开发者，都值得认真看一遍。

1. 并行开发：效率翻倍的正确姿势

同时启动 3-5 个 git worktree（工作树），每个工作树运行各自的 Claude 会话——这是提升效率最有效的方法，没有之一。

Claude Code 团队大多数成员都在用 worktree，而不是多个 git checkout。

实操技巧：
  • 给工作树命名，设置 shell 别名（如 za、zb、zc），一键切换
  • 有一个专门的工作树叫"分析"，只用来读日志和跑 BigQuery
核心逻辑： 让不同的 Claude 实例并行处理不同任务，主会话保持干净，专注于当前最紧急的事。

2. 计划先行：磨刀不误砍柴工

做复杂任务之前，先开 Plan Mode（计划模式），把计划写得越细越好，让 Claude 一次性把代码写对。

有人甚至会这样分工：Claude A 写计划 → Claude B 审查计划 → Claude C 执行。

更关键的一点：遇到问题，不要硬撑着推进，切回计划模式重新规划。 测试验证环节也一样，不要跳过。

反面案例： 很多人一遇到 bug 就开始硬调、调了半天发现方向错了。不如让 Claude 先给你一个排查计划。

3. 沉淀你的 CLAUDE.md

每次纠正 Claude 的错误之后，加一句："更新你的 CLAUDE.md，这样你不会再犯同样的错误。"

Claude 非常擅长为自己编写规则。随着时间推移，毫不留情地迭代 CLAUDE.md，直到你的错误率显著下降。

进阶做法：

  • 让 Claude 为每个项目维护一个笔记目录，每次提交 PR 后更新
  • 把这个笔记目录的路径加到 CLAUDE.md 里，Claude 就能随时调用
本质： 把"踩过的坑"变成"永久记忆"，而不是每次都重新踩。

4. 把重复工作变成 Skills

如果一件事你一天要做不止一次，就把它变成一个 Skill 或斜杠指令，在所有项目中复用。

几个真实的团队内部 tips：
  • 创建一个 /techdebt 斜杠命令，会话结束时自动运行，查找并删除重复代码
  • 设置一个斜杠命令，把 Slack、Google Drive、Asana、GitHub 上 7 天的数据同步到 Claude 的 context
  • 构建"分析工程师风格"的 agent，能写数据库策略模型、审查代码、测试开发环境变更
本质： 用一次工程时间，省掉未来一百次重复操作。

5. 直接说"修"，别啰嗦

遇到 bug，不要事无巨细地指导具体操作。直接说：

  • "修复"
  • "去修那个失败的 CI 测试"
不要给上下文，让 Claude 自己看日志排查。 它在看 docker 日志、排查分布式系统故障方面，出乎意料地强大。

团队的做法是：启用 Slack MCP，把 Slack 上讨论 bug 的内容直接粘贴给 Claude，说一句"修复"，零上下文切换。

核心逻辑： 你不需要是调试专家，Claude 可以是。让它去干活。

6. 让你的提示词更上一层楼

a. 向 Claude 发起挑战：
  • "在我通过你的测试之前别提交 PR"——让它当你的代码审查员
  • "证明给我看，这行得通"——让它对比主分支和特性分支的差异
b. 遇到烂方案时：
  • "鉴于你现在掌握的所有信息，放弃这个方案，实施更优雅的解决方案"
c. 交接工作之前：
  • 编写详细的规范，减少歧义。越具体，输出越好。

7. 终端配置也是战斗力

团队成员非常喜欢 Ghostty——同步渲染、24 位色彩、完善的 Unicode 支持。

提效配置：
  • 用 /statusline 自定义状态栏，始终显示 context 用量和当前 Git 分支
  • 终端标签页颜色编码和命名，或用 tmux——每个任务/工作树对应一个标签页
  • 语音输入：你的语速是打字速度的 3 倍，语音生成的提示词更详细

8. 善用 Subagents

对于需要更多计算资源解决的问题，在请求中添加 "使用 subagents" 参数。

正确的用法：
  • 把单个大任务分解为多个子任务，分给 subagents
  • 主 agent 保持上下文窗口干净，专注于特定任务
  • 通过钩子把权限请求路由到 Opus 4.5，让它扫描攻击并自动批准安全请求

9. 用 Claude 做数据分析

让 Claude Code 使用 bq 命令行工具实时提取和分析指标。团队已经集成 BigQuery skill，所有人都直接在 Claude Code 里做分析查询。

一个数据： 有人已经超过 6 个月没写过一行 SQL 代码了。 本质： 任何有 CLI、MCP 或 API 的数据库都可以这么用。告别手动写 SQL，让 Claude 当你的数据分析师。

10. 和 Claude 一起学习

a. 开启"解释性"输出样式： 让 Claude 解释每一步背后的"为什么"，而不只是"做什么"。 b. 让它做可视化演示： 让 Claude 做一个 HTML 演示文稿来解释你不熟悉的代码——它能做出非常棒的 PPT。 c. 绘制 ASCII 图： 让 Claude 绘制新协议和新代码库的 ASCII 图，帮助你理解复杂逻辑。 d. 填空式学习法： 你先说你的理解，Claude 提后续问题填补知识空白，并记录学习结果。

最后

这 10 条技巧，核心就三句话：

  1. 让 AI 干它擅长的，你干你擅长的
  2. 把经验固化，不要每次都重新积累
  3. 不要手把手教，让它自己看、自己修

Claude Code 不是一个写代码的工具，它是一个能帮你思考、分析、学习的 AI 搭档。用对它，你会发现自己的效率提升不是一点点。
